## 核心规则前提

以下四条是本次复核的硬性前提，所有判定均基于此：

| # | 规则 | 说明 |
|:--:|:-----|:-----|
| R1 | **所有成品无品质分级**：熟水、茶饮、甜品都只有单一稀有度，不存在★/★★/★★★ |
| R2 | **熟水 = 单一种食材制作，一键交付**：每种食材 ↔ 1种独特熟水成品，选食材→一键制作并贩卖，不表现制作过程 |

| R4 | **灵韵纯收集向**：图标共鸣只解锁成就/称号，不加成任何数值 |

## 底卡系统·极速理解指南

> **给第一次接触本方案的人看的3分钟入门**——底卡的规则和花园世界的花瓶完全一样简单。

### 一句话定义

**底卡 = 花瓶，3个槽位放3个材料 = 成品。没有更多规则。**

```
花园世界：  选花瓶 → 3个花槽各插1朵花 → 出花艺作品
梦华小院：  选底卡 → 3个食材槽各放1种食材 → 出茶饮/甜品
```

### 与花园世界 1:1 映射

| 花园世界概念 | 梦华小院对应 | 是否一致 |
|:-----------|:------------|:--------:|
| 花瓶 | **底卡（茶具+底底）** | ✅ 完全一样 |
| 3个花槽 | **3个食材槽位** | ✅ 数量相同 |
| 每槽放1朵花 | **每槽放1种食材** | ✅ 操作相同 |
| 插完出成品 | **放完出成品** | ✅ 逻辑相同 |
| 不同花瓶能插的花不同 | **不同底卡能放的食材不同** | ✅ 约束相同 |

### 玩家操作流程（精轨·做茶饮/甜品）

```
步骤1：选一张底卡
        ┌──────────────────────────────┐
        │  汝窑天青盏 · 龙井底          │  ← 这就是"花瓶"
        │                              │
        │  [槽位1 主鲜果]  ← 放水果     │
        │  [槽位2 辅本草]  ← 放草本    │  ← 三个空槽，跟花瓶一模一样
        │  [槽位3 花卉点缀] ← 放花卉   │
        │                              │
        │  可得组合：27种              │
        └──────────────────────────────┘

步骤2：每个槽位选1种你种的食材
        ┌──────────────────────────────┐
        │  汝窑天青盏 · 龙井底          │
        │                              │
        │  [槽位1 主鲜果]  🍐 雪梨      │  ← 从你的食材库中挑选
        │  [槽位2 辅本草]  🌿 薄荷      │
        │  [槽位3 花卉点缀] 🌸 桂花     │
        └──────────────────────────────┘

步骤3：点"制作" → 出成品 → 收入茶饮图鉴
        （首次解锁该底卡时播放互动式水墨动画）
```

**就是这么简单：选底卡 → 填3个坑 → 点制作。**

### 与花园世界的2处关键差异

| # | 差异点 | 说明 |
|:--:|:------|:-----|
| 1 | **类别约束型槽位** | 花瓶的3个槽通常"什么花都能插"；底卡的3个槽有类型限制——槽1只能放鲜果、槽2只能放本草、槽3只能放花卉。这降低了选择困难（不会在几十种食材里乱翻） |
| 2 | **排他原则** | 每种食材全局只属于一张底卡（比如荔枝只在"T008 荔浆底"的槽位中出现）。保证了收集动力："我种了荔枝，就一定要解锁荔浆底这张底卡来用" |

### 茶饮和甜品用同一套操作

不需要学两套系统。**茶饮底卡和甜品类底卡操作完全一致**：

| 类型 | 底卡示例 | 操作 | 区别仅在成品形态 |
|:-----|:---------|:-----|:----------------|
| 茶饮 | 定窑白瓷盏·酸梅汤底 | 选底卡→填3槽→制作 | 出一杯茶饮 |
| 甜品·冻 | 磁州彩碟·藕粉冻底 | 选底卡→填3槽→制作 | 出一碗甜品冻 |
| 甜品·糕 | 竹笼蒸格·桂花糕底 | 选底卡→填3槽→制作 | 出一块糕点 |

玩家感知到的差异只有两点：
1. **底卡图标不同**（茶具 vs 甜品碟/模具）
2. **成品外观不同**（杯子里的饮料 vs 碗里的甜品）

**底层操作逻辑100%一致。**

### ✅ 问题② 熟水 vs 茶底体系碰撞 → 完美解决

这是本方案的**最大亮点**。

**原担忧**：梦华录一半饮品不是"茶"，茶底体系兜不住。

**解法**：两种订单类型天然分流：

```
梦华录饮品分类              游戏内映射                收集归属
─────────────────────────────────────────────────────────

【熟水 / 饮子】             →  卖熟水（居民订单）      → 食材图鉴（底层）
  单一食材 = 一种独特熟水（一键完成）
  ─────────────────────────────────────────────
  紫苏饮子  ──────────────→  [紫苏叶] → 一键做成→交付
  薄荷饮   ──────────────→  [薄荷叶] → 一键做成→交付
  沉香熟水 ──────────────→  [沉香木]  → 一键做成→交付
  红果饮   ──────────────→  [山楂]    → 一键做成→交付
  桂花饮   ──────────────→  [桂花]    → 一键做成→交付
  香薷饮   ──────────────→  [香薷]    → 一键做成→交付
  
  居民订单可需求多种：
    "我要3杯紫苏饮" "来2碗薄荷汤"
  UI上通过图标/名称强关联表达食材↔熟水映射
  （不表现制作过程——快轨=效率）

【点茶 / 正式茶饮】          →  雅客/宫宴订单（底卡制）  → 🏆茶饮图鉴
  底卡(茶具+底底) + 3槽位食材 = 一个成品
  15-25秒李子柒式调制动画
  ─────────────────────────────────────────────
  北苑先春点茶 ──────────→  [贡品级底卡 + 主鲜果+辅本草+花卉]
  径山茶    ─────────────→  [雅士级底卡 + 同上]

【果子 / 甜品】              →  雅客/宫宴订单（甜品底卡）→ 🏆茶饮图鉴
  ─────────────────────────────────────────────
  碧涧豆儿糕 ────────────→  [甜品·糕类底卡 + 食材]
  桂花茶果  ────────────→  [甜品·糕类底卡 + 桂花等]
  鹿鸣饼    ────────────→  [甜品·糕类底卡]
```

**关键设计决策**：

- 熟水 = **单一种材**，逻辑干净，一一对应
- 熟水**不是独立图鉴**——它是订单的交付载体，玩家收集的是底层食材本身（食材图鉴里的每条都有古雅名字+典籍出处）
- 快轨（卖熟水）= **一键完成无动画**；精轨（做茶饮）= **15-25秒调制动画**。快=效率，精=体验，节奏对比鲜明
- 无品质分级意味着**每种食材和每种茶饮都是唯一的收集条目**

**结论**：✅ 梦华录的全部饮品都能找到归宿，不需要额外新增系统框架。

### ✅ 问题③ 材料品种超纲 → 已被叙事框架合理解释

| 食材类型 | 数量 | 历史态度 | 示例 |
|----------|:----:|:---------|------|

### ✅ 问题⑥ 操作仪式感 → 双轨制已形成天然节奏差

**双轨制的节奏设计本身就是答案**：

| | 🏃 快轨·卖熟水 | 🎬 精轨·做茶饮 |
|---|---|---|
| **操作** | 选食材 → **一键完成** | 选底卡+三槽位 → **15-25秒动画** |
| **制作过程** | ❌ 不表现 | ✅ 李子柒式仪式感 |
| **UI表达** | 图标/名称强关联 | 完整链路动画 |
| **定位** | 效率优先，日常循环 | 体验优先，情感沉浸 |

**这种节奏差的优点**：
- 快的时候真快（日常循环不拖泥带水）
- 慢的时候真慢（享受调制过程的治愈感）
- 两种状态切换，玩家不易疲劳

**最低要求**：精轨必须有 **P0 调制动画**（15-25秒），否则和"填完格子直接出结果"没有区别。由于R1（无品质分级），动画的重点是**过程感**而非**品质揭晓的惊喜感**。

### ✅ 问题⑧ 灵韵/花灵系统 → 下沉适配三层制（2026-04-30更新）

**原问题**：花灵系统玩不明白、门槛过高。

> **⚠️ 重要纠正**：所谓"门槛过高"需要精确界定——花园世界中**免费玩家也有大量获得真花**，参与门槛并不高。真正高的是**完整版的付费获取门槛**（¥1000+全套）。评审方可能混淆了"参与门槛"和"完整版获取门槛"。

**关键数据（内部人士确认，花园世界）**：
- 花灵收入占总收入 **70%**
- 付费用户占比 **< 1%**
- 人均付费 **¥1000+**（完整版花灵）
- 本质：**极端头部集中的"鲸鱼税"模式**

**核心判断**：砍掉花灵=砍掉70%收入引擎；但原版¥1000门槛对下沉用户不可达。
**解法：不是移除，是下沉适配。**

#### 新方案：花灵三层制

| 层级 | 用户群体 | 获取方式 | 权益 | 目标占比 |
|:----:|:---------|:---------|:-----|:--------:|
| **免费层** | 大众用户（90%+）| 每天看广告抽1次 + 每日任务赠送抽卡券 | 纯收集向：灵韵共鸣成就、图鉴 completeness、称号解锁 | 90%+ |
| **VIP层** | 月卡/通行证用户（5-10%）| VIP可额外抽1次 + 月卡送抽卡券 | 轻度数值加成 + 收集加速 + 外观展示 | 5-10% |
| **鲸鱼层** | 大R/鲸鱼用户（<1%）| 中R付费点购买花灵抽卡券 / 完整套直购¥648-1280 | 完整数值加成 + 稀有外观 + 专属称号 + 社交炫耀 | <1% |

#### 与原方案的差异

| 维度 | 旧方案（纯收集向/移除花灵） | 新方案（三层适配制） |
|:-----|:--------------------------|:---------------------|
| 花灵引擎 | 移除或大幅弱化 | **完整保留** |
| 免费用户参与感 | 有（纯收集）| **更强（每天免费抽）|
| 鲸鱼收入 | ❌ 丢失70%主力收入 | ✅ **完整保住70%引擎** |
| VIP价值感 | 弱 | **强（VIP=多一抽硬权益）**|
| 与R1兼容性 | ✅ 完美兼容 | ✅ **完美兼容（大众层无数值）**|

> **设计精髓**：不剥夺任何人参与感（免费每天也能抽）、VIP有明确硬权益（多一抽）、鲸鱼照常交完整套的税（¥648-1280档位保留）。**三方共赢。**

**与文化的结合点**：每种灵韵图标关联传统吉祥纹样（蝙蝠=福、如意=顺），凑齐时按层级解锁不同奖励——大众层得文化称号（"百草通""茶博士"），鲸鱼层额外获得稀有外观和社交炫耀。

### ✅ 问题⑨ 用户下沉市场 → 是机会而非风险

| 原始担忧 | 实际情况 | 判定 |
|----------|---------|------|
| "用户下沉会拉低产品调性" | 下沉市场用户基数远超一二线（~2-3亿 vs ~5000万潜在玩家）| ✅ **机会** |
| "需要防下沉" | 拼多多/李子柒/抖音已经证明下沉用户的消费力和传播力 | ✅ **方向正确** |

**击中下沉市场的正确方法**：

| 方法 | 说明 |
|------|------|
| **文案白话化（P6）** | 操作/提示全大白话："种下去""浇一下""该施肥了" |
| **名字雅致化** | 只有食材名和成品名引经据典：离支、丹若、蕃荷、水芝…… |
| **亲民定价** | ¥6-18明码标价，不搞"首充送VIP"那套 |
| **清晰UI** | 大按钮、大字体、少层级——小镇阿姨也能一眼看懂 |

> **高级感来自"我学到了"，不是来自"我看不懂"。** 文化自信 ≠ 文青，而是让每个学历背景的玩家都能自然接触传统文化。
