核心规则前提

以下四条是本次复核的硬性前提，所有判定均基于此：

# 规则 说明
R1 所有成品无品质分级：熟水、茶饮、甜品都只有单一稀有度，不存在★/★★/★★★
R2 熟水 = 单一种食材制作，一键交付：每种食材 ↔ 1种独特熟水成品，选食材→一键制作并贩卖，不表现制作过程
R3 两层图鉴：🌱食材图鉴（唯一种植收集核心）+ 🏆茶饮甜品图鉴（深度制作）。熟水是订单交付载体，不作为独立图鉴
R4 灵韵纯收集向：图标共鸣只解锁成就/称号，不加成任何数值

逐项问题验证

✅ 问题① 身份矛盾 → 已解决

原始担忧 设计方案 判定
茶坊老板不会自己种地 穿越者人设：带着现代记忆来到宋朝，既种又卖 ✅ 逻辑通顺
行为逻辑撕裂 自产自销模式在古代很常见（前店后园） ✅ 合理
缺乏经营感 卖熟水 + 雅客/宫宴订单 双轨制提供经营反馈 ✅ 有经营循环

补充说明：在UI叙事上强化"后院种植、前厅售卖"的空间关系。穿越者设定同时天然解释了食材多样性——西域/南洋/西洋贸易带来的舶来品。

✅ 问题② 熟水 vs 茶底体系碰撞 → 完美解决

这是本方案的最大亮点。

原担忧：梦华录一半饮品不是"茶"，茶底体系兜不住。

解法：两种订单类型天然分流：

梦华录饮品分类              游戏内映射                收集归属
─────────────────────────────────────────────────────────

【熟水 / 饮子】             →  卖熟水（居民订单）      → 食材图鉴（底层）
  单一食材 = 一种独特熟水（一键完成）
  ─────────────────────────────────────────────
  紫苏饮子  ──────────────→  [紫苏叶] → 一键做成→交付
  薄荷饮   ──────────────→  [薄荷叶] → 一键做成→交付
  沉香熟水 ──────────────→  [沉香木]  → 一键做成→交付
  红果饮   ──────────────→  [山楂]    → 一键做成→交付
  桂花饮   ──────────────→  [桂花]    → 一键做成→交付
  香薷饮   ──────────────→  [香薷]    → 一键做成→交付

  居民订单可需求多种：
    "我要3杯紫苏饮" "来2碗薄荷汤"
  UI上通过图标/名称强关联表达食材↔熟水映射
  （不表现制作过程——快轨=效率）

【点茶 / 正式茶饮】          →  雅客/宫宴订单（底卡制）  → 🏆茶饮图鉴
  底卡(茶具+底底) + 3槽位食材 = 一个成品
  15-25秒李子柒式调制动画
  ─────────────────────────────────────────────
  北苑先春点茶 ──────────→  [贡品级底卡 + 主鲜果+辅本草+花卉]
  径山茶    ─────────────→  [雅士级底卡 + 同上]

【果子 / 甜品】              →  雅客/宫宴订单（甜品底卡）→ 🏆茶饮图鉴
  ─────────────────────────────────────────────
  碧涧豆儿糕 ────────────→  [甜品·糕类底卡 + 食材]
  桂花茶果  ────────────→  [甜品·糕类底卡 + 桂花等]
  鹿鸣饼    ────────────→  [甜品·糕类底卡]

关键设计决策：

  • 熟水 = 单一种材，逻辑干净，一一对应
  • 熟水不是独立图鉴——它是订单的交付载体，玩家收集的是底层食材本身（食材图鉴里的每条都有古雅名字+典籍出处）
  • 快轨（卖熟水）= 一键完成无动画；精轨（做茶饮）= 15-25秒调制动画。快=效率，精=体验，节奏对比鲜明
  • 无品质分级意味着每种食材和每种茶饮都是唯一的收集条目

结论：✅ 梦华录的全部饮品都能找到归宿，不需要额外新增系统框架。

✅ 问题③ 材料品种超纲 → 已被叙事框架合理解释

原始担忧 设计方案 判定
宋代没有番茄/辣椒等美洲作物 穿越者 + 丝路贸易：西域/南洋/西洋来物，天然合理 ✅ 解决
514种远超宋代实际品种 四级稀有度中上古传说级本身就是非历史向 ✅ 可接受

各级食材的历史边界策略：

食材类型 数量 历史态度 示例
市井级食材 ~172种 尽量宋/明清已有作物，保持历史质感 荔枝、枇杷、菱角、莼菜
雅士级食材 ~145种 引入域外品种（丝绸之路/海外贸易品） 葡萄（西域）、芒果（南洋）
贡品级食材 ~110种 可以夸张但要有"贡品"叙事支撑 龙眼、人参、燕窝
上古传说级食材 ~87种 完全放开，仙草神果 灵芝、何首乌、千年参

不需要完全符合史实。游戏的文化内涵体现在对茶文化、宋元时期花卉/水果/本草的了解，以及典籍中名字和出处的展示——这是文化自信的体现（边玩边涨知识），不是历史考据题。

设计方案已覆盖大部分：

梦华录果子 底卡映射 状态
碧涧豆儿糕 甜品·糕 类底卡（5张） ✅
桂花茶果 甜品·糕 类底卡 + 桂花 ✅
鹿鸣饼 甜品·糕 类底卡 ✅

梨条桃圈 同上 ❌ 缺口
越梅蜜饯 同上 ❌ 缺口

唯一缺口：蜜饯类果子目前没有对应的底卡。现有甜品大类： - 甜品·冻（10张） - 甜品·糕（5张） - 甜品·布丁/冰品（4张）

建议：增加 甜品·蜜饯 子类（2-3张底卡），覆盖蜜饯雕花、梨条桃圈、越梅蜜饯。

✅ 问题⑥ 操作仪式感 → 双轨制已形成天然节奏差

双轨制的节奏设计本身就是答案：

🏃 快轨·卖熟水 🎬 精轨·做茶饮
操作 选食材 → 一键完成 选底卡+三槽位 → 15-25秒动画
制作过程 ❌ 不表现 ✅ 李子柒式仪式感
UI表达 图标/名称强关联 完整链路动画
定位 效率优先，日常循环 体验优先，情感沉浸

这种节奏差的优点： - 快的时候真快（日常循环不拖泥带水） - 慢的时候真慢（享受调制过程的治愈感） - 两种状态切换，玩家不易疲劳

最低要求：精轨必须有 P0 调制动画（15-25秒），否则和"填完格子直接出结果"没有区别。由于R1（无品质分级），动画的重点是过程感而非品质揭晓的惊喜感。

✅ 问题⑦ 核心收集对象变更 → 方案清晰且改动更小

从花园世界的收集体系到梦华小院：

维度 花园世界 梦华小院 变化评估
收集对象① 花卉 🌱食材（鲜果/花卉/本草） 扩容（1类→3类）
收集对象② 花艺作品 🏆茶饮/甜品成品 直接映射
图鉴数量 2个（花卉图鉴+花艺图鉴） 2个（食材图鉴+茶饮图鉴） 结构不变
快速行为 卖花 卖熟水（一键） 行为替换
深度制作 做花艺 做茶饮/甜品（底卡+三槽） 机制映射

关键认知：这不是什么大重构——花园世界本来就是2个图鉴，梦华小院也是2个，直接1:1对应。数据结构基本不变，主要是内容扩容和美术替换。

影响维度 评估
数据结构变化 🟡 中等（字段映射：花瓶→底卡、花槽位→食材槽位）
图鉴系统 ✅ 最小改动（2→2直接映射）
订单校验 🟡 小-中（文本替换+双轨产出逻辑适配）
美术资源增量 🔴 主要工作量（~69张底卡图 + ~514个食材图标 + 成品零件拼接系统）
品质可视化 ✅ 不需要（R1确认无品质分级，大幅减少工作量）
库存UI 🟡 小（三分栏：食材/底卡/成品缓存）

结论：比最初预估的工作量少约40%（主要省掉了品质分级相关的全部设计和资源）。图鉴结构不变是最大的简化点。

✅ 问题⑧ 灵韵系统改造 → 方案已确定

原问题：花灵系统玩不明白、门槛过高。

新方案：图标共鸣法 · 纯收集向

维度 原版（问题） 新版（解决）
作用 影响品质星级 纯收集：凑同灵韵→成就/称号
付费门槛 ~1000元 6元起步
复杂度 高（属性计算） 低（颜色匹配）
与R1兼容性 依赖品质分级存在 ✅ 完美兼容（不加数值）

为什么这个方案特别适合当前架构：

因为R1规定"无品质分级"，所以任何基于数值加成的灵韵方案都会产生矛盾。而纯收集向方案巧妙地绕开了这个问题——灵韵变成了"图鉴之上的图鉴"，一个额外的探索维度，不影响任何已有数值体系。

与文化的结合点：每种灵韵图标可以关联一种传统吉祥纹样（如蝙蝠=福、如意=顺），凑齐时解锁的不是数值加成，而是文化称号（如"百草通""茶博士""花神使者"），进一步强化文化自信。

✅ 问题⑨ 用户下沉市场 → 是机会而非风险

原始担忧 实际情况 判定
"用户下沉会拉低产品调性" 下沉市场用户基数远超一二线（~2-3亿 vs ~5000万潜在玩家） ✅ 机会
"需要防下沉" 拼多多/李子柒/抖音已经证明下沉用户的消费力和传播力 ✅ 方向正确

击中下沉市场的正确方法：

方法 说明
文案白话化（P6） 操作/提示全大白话："种下去""浇一下""该施肥了"
名字雅致化 只有食材名和成品名引经据典：离支、丹若、蕃荷、水芝……
亲民定价 ¥6-18明码标价，不搞"首充送VIP"那套
清晰UI 大按钮、大字体、少层级——小镇阿姨也能一眼看懂

高级感来自"我学到了"，不是来自"我看不懂"。 文化自信 ≠ 文青，而是让每个学历背景的玩家都能自然接触传统文化。
